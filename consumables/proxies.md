# Прокси

## Зачем

Каждый запрос к API проекта идёт с твоего IP. Если 100 кошельков ходят с одного IP — это **сибил**, проект банит акки или вообще обнуляет результаты.

Прокси решают эту проблему: каждому кошельку (или группе из 1–3) — свой IP.

## Где брать

[**maxy.tools**](https://maxy.tools/) → раздел «Прокси».

> Это проверенный сервис от друга. Цены адекватные, поддержка живая.

Какие типы прокси бывают:

| Тип | Скорость | Анти-детект | Цена | Под что |
|---|---|---|---|---|
| **Datacenter (DC)** | ⚡⚡⚡ | ⭐⭐ | $ | Лёгкие задачи, не палевные API |
| **ISP / Static residential** | ⚡⚡ | ⭐⭐⭐⭐ | $$ | Большинство тестнетов и фарма |
| **Residential (rotating)** | ⚡ | ⭐⭐⭐⭐⭐ | $$$ | Жёсткие анти-сибил-фильтры |
| **Mobile** | ⚡ | ⭐⭐⭐⭐⭐ | $$$$ | Самые палевные кейсы |

Дефолт под софты из этого хаба — **ISP / static residential**, по 1 прокси на 1–3 кошелька.

## Формат

В софтах используется формат:

```
http://username:password@host:port
```

Один прокси — на одну строку в `input/proxies.txt`.

Также поддерживается SOCKS5:

```
socks5://username:password@host:port
```

## Сколько прокси

| Акков | Прокси | Соотношение |
|---|---|---|
| 1–10 | 1–3 | по 3–4 акка на прокси (ничего страшного) |
| 10–50 | 5–15 | по 2–3 акка на прокси |
| 50–500 | 30–200 | по 1–2 акка на прокси |
| 500+ | 1:1 | строго один прокси на один акк |

## Проверка

В `parameters.py` Outcome-MXW есть флаг:

```python
CHECK_PROXIES_ON_START = False
```

Поставь `True` — софт перед стартом проверит каждый прокси и заменит/выкинет нерабочие. Полезно при первом запуске.

## Если прокси падает в процессе

`core/proxy_manager.py` сам ловит ошибки прокси и переключает на следующий из списка. Главное — иметь запас.

## Чек-лист по прокси

* [ ] Куплены статик ISP (для Outcome-MXW и подобных)
* [ ] Положены в `input/proxies.txt` в формате `http://user:pass@host:port`
* [ ] Соотношение к кошелькам — 1:1–1:3
* [ ] Регион/страна прокси соответствует ожиданиям проекта (если есть гео-чувствительность)

# Анатомия типового софта

Все софты из хаба построены по одной схеме. Если разобрался с одним — поймёшь любой.

## Структура папок

```
outcome-MXW/
├── core/              # Базовые компоненты (общие для всех модулей)
│   ├── account.py         # Класс Wallet
│   ├── outcome_client.py  # HTTP-клиент проекта + авторизация
│   ├── database.py        # Шифрованная БД приватников
│   ├── proxy_manager.py   # Менеджер прокси
│   ├── capsolver.py       # Антикапча
│   ├── logger.py          # Логирование
│   └── utils.py           # Утилиты (паузы, ретраи)
├── modules/           # Логика конкретных активностей
│   ├── faucet.py          # Получение тестовых токенов
│   ├── betting.py         # Ставки на события
│   ├── sell_positions.py  # Закрытие позиций
│   └── wallet_stats.py    # Парсинг статистики
├── input/             # Твои данные (НЕ ПУШИТСЯ В ГИТХАБ)
│   ├── private_keys.txt   # Приватники (очищается после создания БД)
│   ├── proxies.txt        # Прокси
│   ├── database.enc       # Зашифрованная БД (создаётся автоматически)
│   └── database.salt      # Соль для шифрования
├── logs/              # Логи запусков (создаётся автоматически)
├── env/               # Виртуальное окружение (создаётся через python -m venv)
├── parameters.py      # ВСЕ настройки софта (главный файл для тебя)
├── main.py            # Точка входа, меню
├── requirements.txt   # Зависимости Python
├── README.md          # Краткое описание
└── .gitignore         # Что не пушить в git
```

## С чем работаешь ты

| Файл                     | Зачем                                                                            |
| ------------------------ | -------------------------------------------------------------------------------- |
| `parameters.py`          | **Главное.** Все настройки софта — тут. CapSolver-ключ, потоки, суммы, задержки. |
| `input/private_keys.txt` | Сюда кладёшь приватники, **только перед** созданием БД.                          |
| `input/proxies.txt`      | Прокси, тоже в начале.                                                           |

Всё остальное (`core/`, `modules/`, `main.py`) — лучше не трогать, если не понимаешь, что делаешь.

## С чем работает софт

| Файл/папка                | Что                                                       |
| ------------------------- | --------------------------------------------------------- |
| `input/database.enc`      | Здесь живут зашифрованные приватники после создания БД.   |
| `logs/bot_YYYY-MM-DD.log` | Сюда пишутся логи каждого запуска. Смотри их при ошибках. |
| `core/proxy_manager.py`   | Управляет прокси, выкидывает мёртвые.                     |

## `parameters.py` — что обычно настраивают

* `CAPSOLVER_API_KEY` — твой ключ от антикапчи.
* `THREADS` — сколько кошельков обрабатывать одновременно (10–20 норм для большинства).
* `*_RANGE` — диапазоны рандомизации (суммы, задержки). Не делай слишком узко — палится.
* `RETRY_*` — сколько раз ретраить при ошибке.

## Логи

В каждом запуске софт создаёт файл `logs/bot_YYYY-MM-DD.log`. Структура строки:

```
2026-04-29 10:24:30 | INFO | 0x5236...b4EE | Начинаем обработку | proxy: http://...
```

| Часть                        | Что                                      |
| ---------------------------- | ---------------------------------------- |
| `2026-04-29 10:24:30`        | Время                                    |
| `INFO/SUCCESS/ERROR/WARNING` | Уровень                                  |
| `0x5236...b4EE`              | Кошелёк (или `MAIN` для общих сообщений) |
| Текст                        | Что произошло                            |

При ошибке ищи `ERROR` в логе — обычно сразу видно, в чём дело.

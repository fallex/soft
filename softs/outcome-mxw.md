# OUTCOME-MXW

> 🔗 **Гитхаб:** [github.com/fallex/outcome-MXW](https://github.com/fallex/outcome-MXW)

Бот для автоматизации [Outcome.xyz](https://testnet.outcome.xyz) — тестнет prediction-market платформы HYPE.

## Что делает

1. **Авторизация SIWE** — Sign-In with Ethereum, получает сессию.
2. **Фаусет** — забирает тестовый USDT0 (POST `/faucet/mint`, до 10 000 за раз).
3. **Ставки** — открывает позиции BUY YES/NO на рандомные или конкретные события.
4. **Холд + продажа** — держит позиции рандомное время, потом закрывает.
5. **Принудительное закрытие** — режим экстренной продажи всех открытых позиций.
6. **Парсинг кошельков** — собирает статистику.

## Полный цикл одного аккаунта

```
SIWE auth → enable trading → faucet (8500–9800 USDT0) →
N ставок (10–20, рандом по событиям) → холд 60–300 сек на позицию →
продажа → отмена остатка → финальный баланс
```

## Настройка

### 1. Расходники

* **Прокси** — [maxy.tools](https://maxy.tools/) → static ISP.
* **Капсолвер** — [регистрация](https://dashboard.capsolver.com/passport/register?inviteCode=ypCnEHZAzZeh), пополнить $5+, скопировать API key.
* **Кошельки** — генерируй сам (см. [Расходники → Кошельки](../consumables/wallets.md)).

### 2. Файлы input

В сайдбаре Cursor (или просто в Finder/Explorer) переименуй файлы в папке `input/`:

* `private_keys.txt.example` → `private_keys.txt`
* `proxies.txt.example` → `proxies.txt`

Или через терминал:

```bash
mv input/private_keys.txt.example input/private_keys.txt    # Mac/Linux
mv input/proxies.txt.example      input/proxies.txt
```

```powershell
move input\private_keys.txt.example input\private_keys.txt   # Windows
move input\proxies.txt.example      input\proxies.txt
```

Открой оба файла в Cursor и впиши приватники / прокси — по одному на строку.

* `input/private_keys.txt` — приватники, по одному на строку.
* `input/proxies.txt` — прокси `http://user:pass@host:port`, по одной на строку.

### 3. parameters.py — обязательные

```python
CAPSOLVER_API_KEY = "CAP-..."  # обязательно
THREADS           = 12         # потоки (10–20 для среднего ПК)
```

### 4. parameters.py — рекомендуемые дефолты

Файл уже идёт с разумными дефолтами, но советую поиграться с настройками, чтобы отличаться от других юзеров.

| Параметр                        | Дефолт         | Описание                                          |
| ------------------------------- | -------------- | ------------------------------------------------- |
| `WALLET_DELAY_RANGE`            | `[10, 30]`     | Рандомная задержка между стартами кошельков (сек) |
| `RETRY_RANGE`                   | `[2, 4]`       | Сколько раз ретраить при ошибке                   |
| `RETRY_DELAY_RANGE`             | `[5, 15]`      | Пауза между ретраями (сек)                        |
| `FAUCET_AMOUNT_RANGE`           | `[8500, 9800]` | Сколько USDT0 запрашивать у фаусета (макс 10 000) |
| `BET_COUNT_RANGE`               | `[10, 20]`     | Сколько ставок за один цикл                       |
| `BET_MIN_AMOUNT`                | `100`          | Минимальная сумма одной ставки (USDT0)            |
| `BET_SIDE`                      | `"RANDOM"`     | YES / NO / RANDOM                                 |
| `BET_SLIPPAGE`                  | `0.05`         | Слиппейдж (5%)                                    |
| `TARGET_EVENT_ID`               | `0`            | Ставить на конкретное событие (`0` = рандом)      |
| `TARGET_CATEGORY`               | `""`           | CRYPTO / STOCK / SPORTS / `""` (все)              |
| `BET_EVENT_MIN_HOURS_TO_EXPIRY` | `48`           | Не ставить на события с резолвом раньше N часов   |
| `HOLD_TIME_RANGE`               | `[60, 300]`    | Холд позиции, сек                                 |
| `SELL_SLIPPAGE`                 | `0.10`         | Слиппейдж при продаже (10%)                       |

## Запуск

```bash
python main.py
```

Меню (стрелки ↑↓ + Enter):

| Пункт                                   | Что делает                                                                                                                   |
| --------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| **Создать базу данных**                 | Шифрует приватники из `input/private_keys.txt` паролем в `input/database.enc`. После — txt очищаются. **Делается один раз.** |
| **Полный цикл**                         | Авторизация → фаусет → ставки → холд → продажа. **Дефолтный режим.**                                                         |
| **Только активности**                   | Без фаусета (если уже есть USDT0 на балансе)                                                                                 |
| **Только пополнение**                   | Только фаусет, без ставок                                                                                                    |
| **Продать все позиции (принудительно)** | Закрывает всё открытое. Аварийный режим.                                                                                     |
| **Парсинг кошельков**                   | Собирает статистику, не делает транзакций                                                                                    |

## Рекомендации по запуску

* **1–2 раза в день** — оптимальная частота для тестнета. Чаще — палевно.
* **«Полный цикл»** — основной режим.
* **«Парсинг»** — раз в неделю проверять, что у акков всё ок.
* **«Продать все»** — если что-то пошло не так и нужно срочно закрыть позиции.

## Что софт умеет автоматически

* Перемешивает порядок кошельков (`SHUFFLE_WALLETS = True`).
* Рандомизирует все задержки.
* При ошибках прокси — переключается на следующий из пула.
* Ретраит запросы с экспоненциальной задержкой.
* Пропускает фаусет, если на балансе уже > 10 000 USDT0.
* Пропускает события, которые истекают раньше `BET_EVENT_MIN_HOURS_TO_EXPIRY`.

## Ошибки и логи

* Все логи: `logs/bot_YYYY-MM-DD.log`.
* В терминале — цветной вывод по уровням (INFO/SUCCESS/WARNING/ERROR).
* Если что-то падает массово — копируй лог-фрагмент в чат MXW.

См. также [FAQ → Outcome](../getting-started/faq.md#softrunsoft-специфические).

## Безопасность

* Приватники шифруются в `input/database.enc` (AES, ключ деривится из пароля)
* Файлы `private_keys.txt` и `proxies.txt` очищаются после создания БД.
* Лог-файлы содержат **последние 4 + 4 символа адреса** (`0x5236...b4EE`), без полных адресов и приватников.
* Сессия авторизации (cookie) в RAM, не сохраняется на диск.

> Подробнее: [Безопасность](../security/).

## Известные особенности Outcome.xyz

* API на тестнете — `https://api-testnet-v2.outcome.xyz/api/v1/outcome`.
* Cloudflare Turnstile **может** просыпаться при подозрительной активности — софт автоматически решает через CapSolver.
* События резолвятся по реальному миру (крипта, спорт, акции). Иногда смещаются дедлайны — софт пропускает протухшие.
* В `enable-trading` есть тонкость с двухфазной подписью — софт это умеет.

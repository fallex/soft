# FAQ

## Cursor IDE

### Зачем нужна IDE, если можно из терминала

Можно и из терминала — никто не запрещает. Но если ты новичок:

* IDE (рекомендую [Cursor](https://cursor.sh/)) подсвечивает синтаксис, находит опечатки, помогает редактировать `parameters.py`.
* В Cursor встроен **AI-помощник**. Бесплатного хватит, чтобы объяснить любую ошибку, подсказать команду или найти строку в коде.
* Встроенный терминал — там же, не надо переключаться между окнами.

### Как открыть встроенный терминал в Cursor

* **Mac:** ⌃ + \` (Ctrl + бэктик) или **View → Terminal**.
* **Win:** Ctrl + \` или **View → Terminal**.

### Как открыть AI-чат в Cursor

* **Mac:** ⌘ + L
* **Win:** Ctrl + L

Вставляй туда ошибку из терминала или выдели кусок кода → задай вопрос.

### AI Cursor говорит «закончились запросы»

Бесплатный план Cursor имеет лимиты на премиум-модели. Когда они кончаются — переключается на бесплатный fallback (gpt-4o mini или подобный). Базовых задач хватает. Если тебе нужно больше - Cursor Pro около $20/мес.

### Cursor не видит мой Python / venv

Открой папку проекта (**File → Open Folder**), затем:

* Нажми **⌘ / Ctrl + Shift + P** → набери `Python: Select Interpreter` → выбери тот, что в `env/bin/python` (Mac) или `env\Scripts\python.exe` (Win).

После этого AI и подсветка кода будут понимать установленные библиотеки.

## Установка

### `python: command not found`

* **Mac:** используй `python3` вместо `python`. Или поставь `brew install python@3.12`.
* **Windows:** Python не в PATH. Переустанови с галочкой **Add to PATH**.

### `pip install` падает с SSL-ошибкой

```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
```

### `ModuleNotFoundError: No module named ...`

Скорее всего venv не активирован. Активируй:

* Mac: `source env/bin/activate`
* Win: `env\Scripts\activate`

Слева должно появиться `(env)`. Потом `pip install -r requirements.txt`.

### PowerShell не даёт активировать venv (`...running scripts is disabled...`)

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

Соглашайся, перезапусти PowerShell.

## Запуск

### «База данных не найдена»

В меню выбери **«Создать базу данных»**, заполнив `input/private_keys.txt` и `input/proxies.txt`. После создания БД — txt очистятся, дальше всё через зашифрованную БД.

### Забыл пароль от БД

БД восстановить нельзя - это AES-шифрование. Удали `input/database.enc` и `database.salt`, заново создай БД с приватниками.

### Меню стрелками не работает

Скорее всего терминал не поддерживает raw mode. Альтернатива: при запуске софт сам предложит ввести номер пункта вручную (введи `1`, `2`, ...).

## Прокси

### Все прокси падают

* Проверь формат: `http://user:pass@host:port`. Никаких пробелов и переносов строк.
* Сделай в `parameters.py`: `CHECK_PROXIES_ON_START = True` — софт проверит и выкинет мёртвые.
* Скорее всего прокси-провайдер забанил тебя — обратись в поддержку у [maxy.tools](https://maxy.tools/).

### Один кошелёк всё время фейлится

* Прокси у этого кошелька «грязный». В логе будет `Connection refused` или таймауты.
* Замени прокси для этого конкретного акка — поправь строку в БД (нужно пересоздать БД с новым `proxies.txt`).

## CapSolver

### `CapSolver баланс: $0.0000`

Пополни баланс на [dashboard.capsolver.com](https://dashboard.capsolver.com/). Минимум $5–10.

### Капсолвер не решает капчу

* Проверь, что ключ скопирован полностью — формат `CAP-XXXX...` (40+ символов).
* Проверь баланс.
* Если ничего — пиши на capsolver-саппорт, у них живой чат.

## Софт-специфические

### Outcome: «Авторизация не удалась»

* Проверь подключение к API: `curl https://api-testnet-v2.outcome.xyz/api/v1/outcome/health`
* Прокси может блочить SIWE-запрос. Попробуй другой прокси.
* Если массово, то вероятно, сервер Outcome упал (бывает на тестнетах).

### Фаусет вернул ошибку

Outcome даёт максимум 10000 USDT0 за раз и имеет cooldown. Если уже получал недавно - пропусти модуль faucet, выбери в меню «Только активности».

## Общее

### Сколько кошельков можно крутить параллельно

* Зависит от мощности ПК и количества прокси.
* На обычном ноуте — 12–20 потоков (`THREADS = 12` в `parameters.py`) — норм.
* Если RAM забита — снизь до 5–8.

### Логи занимают много места

Удаляй старые: `rm logs/bot_2026-03-*.log` (Mac) или `del logs\bot_2026-03-*.log` (Win).

### Где смотреть, что софт реально делает

`logs/bot_YYYY-MM-DD.log` — самый свежий файл за сегодня. Смотри строки с `ERROR` и `WARNING`.

# 🚀 Обзор

## Главное

Все софты в этом хабе — на **Python 3.10+**. Запускать их можно как угодно: чисто из терминала, из любой IDE — это просто скрипты на питоне.

> **Совет:** если ты опытный и привык к своему сетапу (PyCharm, VS Code, голый терминал, vim, что угодно) — работай как привык, дальше можешь не читать.
>
> **Если ты новичок** — настоятельно рекомендую использовать [**Cursor IDE**](https://cursor.sh/).
>
> Это форк VS Code со встроенным **AI-помощником**. Бесплатный план у него слабый, но **с простыми задачами справится**: подскажет команду, объяснит ошибку, найдёт строку в коде. На бесплатке хватит, чтобы пройти всю установку и не запутаться. И — что важно — IDE сама подсветит синтаксис, найдёт опечатку, откроет встроенный терминал в один клик.

## Шаги (общие для всех софтов)

1. [Поставить Cursor IDE](install-mac.md) (Mac) или [то же для Windows](install-windows.md).
2. Установить Python (если ещё не стоит).
3. Зайти на страницу нужного софта в разделе [📦 Софты](../softs/index.md) и скопировать команду клонирования.
4. Через Cursor клонировать репо и открыть папку проекта.
5. В встроенном терминале Cursor создать виртуальное окружение и поставить зависимости.
6. Открыть `parameters.py` в Cursor, заполнить поля (расходники, потоки, параметры).
7. Подготовить `input/` — приватники, прокси.
8. Запустить `main.py` — кнопкой ▶ или из терминала.

> Все шаги одинаковые **для любого** софта в хабе. Что специфично — берётся со страницы софта (URL репо, описание параметров, формат input).

## Зачем виртуальное окружение

Виртуальное окружение (`venv`) — это изолированная папка с библиотеками. Зачем:

* У каждого софта свои версии зависимостей.
* Не засоряет системный Python.
* Удалить софт = удалить папку.

**Без venv будут конфликты версий.** Не пропускай этот шаг — Cursor покажет команды, надо их выполнить.

## Один Python — много софтов

Можно установить Python один раз и использовать его для всех софтов из хаба. Главное — для каждого софта **свой `env`** (виртуалка) внутри его папки.

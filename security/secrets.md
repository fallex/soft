# Что НЕЛЬЗЯ хранить

## Жёсткое правило

> **Сид-фразы и приватные ключи от основных кошельков НЕ должны лежать на компьютере. Никогда. Ни в каком виде.**

«В каком виде» означает:

* ❌ В `.txt` файле (даже на рабочем столе, даже в скрытой папке).
* ❌ В заметках macOS / Windows.
* ❌ В пароле менеджере, который синкается через облако.
* ❌ В `.env` файле проекта.
* ❌ В переписке Telegram «себе».
* ❌ В черновике почты.
* ❌ В скриншоте, который попал в облако.
* ❌ В git-коммите (даже приватного репо — потом случайно сделаешь публичным).

## Где хранить сид-фразы

| Способ | Безопасность | Когда использовать |
|---|---|---|
| Бумага в сейфе | ⭐⭐⭐⭐ | Дефолт. Просто, дёшево, надёжно. |
| Металлическая пластина (CryptoSteel и аналоги) | ⭐⭐⭐⭐⭐ | Если боишься пожара/потопа. |
| Hardware wallet (Ledger, Trezor) | ⭐⭐⭐⭐⭐ | Для активного использования. |
| Шамировы доли (3-of-5 частей) | ⭐⭐⭐⭐⭐ | Для серьёзных сумм, у параноиков. |
| Менеджер паролей (1Password) с 2FA + сложный мастер-пароль | ⭐⭐⭐ | На свой страх и риск. Удобно, но компромисс. |

## Приватники для софтов — как с ними быть

Софты в этом хабе работают по одной из двух моделей:

### Модель 1 — Зашифрованная БД (как в Outcome-MXW)

1. Кладёшь приватники в `input/private_keys.txt`.
2. Запускаешь софт → меню → **«Создать базу данных»**.
3. Софт спрашивает пароль. Ключи шифруются в `input/database.enc`.
4. **Файл `private_keys.txt` автоматически очищается.**
5. Каждый запуск софт спрашивает пароль и расшифровывает БД в RAM.

> Файл `database.enc` без пароля бесполезен. Можно бэкапить.

### Модель 2 — Прямое чтение из txt (старые софты)

Пока используешь софты из этого хаба — у нас всё на модели 1.

## Если приватник всё-таки засветился

1. **Немедленно** перевести всё с этого кошелька на новый.
2. Кошелёк больше не использовать ни для чего.
3. Если приватник попал в публичный гитхаб — считать кошелёк мёртвым. Боты вычистят за минуты.

## Чек-лист «хранения нет»

* [ ] В рабочей директории нет файлов с приватниками вне `input/database.enc`
* [ ] `git status` чистый, ничего «случайно отслеживаемого»
* [ ] Сид основной — оффлайн, на бумаге/металле/хардваре
* [ ] В буфере обмена не висит приватник (после копирования — `pbcopy <<< "clear"` на Mac или просто скопируй что-нибудь нейтральное)
